Unique visitors:
----------------
This information can be found in the first column of each row corresponding to the page requested. In our case, The webpage localhost/conspiracy was visited by the localhost as well as a computer with IP 192.168.1.5. Hence, 2 unique visitors. We can automate this process
by using awk, uniq and wc commands in a pipeline to count the unique visitors. we have to extract only the first column and make sure that we display only unique values and count the number of such values whcih gives the unique visitor count.

>cat access.log | awk "{print $1}" | sort | uniq | wc -l

Number of visits:
-----------------
According to standard log format "%h %l %u %t \"%r\" %>s %b \"%{Referer}i\" \"%{User-agent}i\"" which apache access log seems to follow, %>s is the status code which the server sends back to the client. If it is 200 then everything is OK.
%{Referer} refers to the page from where the current URL is accessed. A value "-" means, the URL was not linked from anywhere but typed explicitly by the user. Hence using awk and obtaining the fields for these 2 values to be 200 and "-" would mean
successful visit. The count of such 200 "-" values will be the number of visits.

>cat access.log | awk "{print $9,$11}" | awk "/200/ && /\"-\"/" | wc -l

Pages:
-------
The number of HTML, php etc. pages totally viewed ( excluding files like img files, css, js etc.). The %{referer} field in the apache log format corresponds to the page through which the current URL is accessed. Using awk or any other similar tool to extract the contents of that column and counting the unique occurrances in that column will yield 
the total number of pages(excluding files which cannot be considered as pages) visited by all visitors. For example, in this line GET /conspiracy/images/newhangar.png HTTP/1.1" 200 33621 "http://localhost/conspiracy/seriesofevents.html", the newhangar.png was fetched/accessed from http://localhost/conspiracy/seriesofevents.html, the %{Referer} field.

>cat access.log | awk "{print $11}" | sort | uniq  |wc -l

Hits:
-----
Any files ( not just pages, but also files like img, js, css etc.) requested from the server. This imformation can be obtained by parsing the "%r" field in the access.log. This is the request line from the client requesting a file. 
For example, in the access.log entries above, the field "GET /conspiracy_area51/images/beigebg.png" implies that beigebg.png file is requested by the client. If we extract this field from the log file and count the number of lines, it gives the total number of all files requested by all users.

>cat access.log | awk "{print $7,$9}" | awk "{print $1, $2 ~ /^200$/}" | grep ^\/ | sort | uniq

Bandwidth:
----------
The total number of bytes returned by the server in response to client's request. The field next to %>s is %b which means the number of bytes returned is specified next to the status code. Extracting only these 2 fields and adding up the values in the %b field gives the bandwidth.
For example, in the line, GET /conspiracy/seriesofevents.html HTTP/1.1" 200 11055, the needed fields are "200 11055" which means 11055 bytes were returned successfully. Adding all such values in all the lines(with successful responses) gives the bandwidth.

Monthly History:
-----------------
The 4th column in the access.log gives the date, month, year and time when the request was received. By accessing rows corresponding to a particular month and calculating the number of visits, pages, hits and bandwidth values (using methods mentioned above), the monthly history can be calculated.

Days of month:
---------------
The 4th column also has the day details. For example, [20/Sep/2016:00:09:10 -0700]. By parsing the individual day data for each month and applying the above methods to calculate the number of visits, pages, hits and bandwidth values, the days of the month usage metrics can be calculated.

Days of week:
--------------

Hours:
-------
The 4th column in access.log can be parsed using ":" delimiter and the hits for the hours can be calculated.

Countries:
------------
The IP addresses can be accessed by extracting the first field from access.log and using the geoiplookup command to find out the country to which it belongs. By using sort and uniq -c command, we can find out the total number of hits for each country and the total number of pages can be found by using the referer column information(in order to 
exclude files that are not pages).
